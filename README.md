# Role Memory Forge v0.4.2

SillyTavern 角色卡记忆插件：分层总结、实时填表、关系网络图、世界书同步、历史聊天补录、走马灯回顾。

## v0.4.2 修复重点

1. **历史聊天扫描重写加强**
   - 不再只依赖 `mes / is_user`。
   - 新增 DOM 文本兜底读取：旧聊天在 `ctx.chat` 里显示为空，但页面上能看到文字时，也能扫描。
   - 新增极限顺序兜底：导入记录被识别成 system/empty 时，也会按实际可读楼层顺序配对。
   - 扫描弹窗会显示：对象文本、DOM 补全文本、可读文本楼层、最终可补录轮数。

2. **历史补录遇到模型空回复不再整段失败**
   - 单轮模型空回复 / 非 JSON 时，会自动用原文生成兜底简记。
   - 阶段总结和大总结模型失败时，也会用已有简记做兜底压缩，避免 200 层只成功几层。

3. **新增“总结后隐藏旧简记”**
   - 阶段总结覆盖过的每轮简记，会从显示区和世界书“每层简记流水”里自动隐藏。
   - 已写进聊天末尾的 RMF 折叠简记，也会在进入阶段总结后从消息里移除。
   - 原始记录仍保存在 JSON_RAW，不会丢失。

4. **Index 数据表兼容增强**
   - 原本的 `<RMF_INDEX_TABLE>` 继续保留。
   - 新增 `TavernDB-ACU-RMF-IndexData` 世界书条目，使用 shujuku / TavernDB-ACU 更接近的 `content` 二维表 JSON 结构。
   - 可在面板里开关：`同步 shujuku/TavernDB-ACU 兼容 Index 数据表`。

## 安装

把本仓库上传到 GitHub，确认根目录包含：

```text
manifest.json
index.js
style.css
README.md
LICENSE
```

然后在 SillyTavern：

```text
扩展程序 → 安装扩展程序 → 粘贴 GitHub 仓库地址
```

更新旧版后建议：

```text
电脑 Ctrl + F5 强刷
手机清缓存后重新进入酒馆
```

## 使用建议

导入别人的旧聊天记录后：

1. 打开 Role Memory Forge 悬浮球。
2. 开启 `启用记忆`。
3. 点 `一键记忆历史聊天记录`。
4. 看弹窗里的 `可读文本楼层` 和 `即将补录轮数`。
5. 如果 200 多层聊天，正常应接近 100 轮左右，具体取决于开场白、系统消息、隐藏消息数量。
